# Reports

Weekly AI-visibility → revenue attribution reports generated by `/peec-report`.

**Filename convention:** `YYYY-Www.md` following ISO week (e.g. `2026-W17.md` for the week of 20 – 26 April 2026).

**Structure of each report:**

1. **Hero + at-a-glance KPIs** — the money quote on top (e.g. "chatgpt.com cohort reached paid this week"), SoV / cite-view-ratio / AI-paid / measurable-origin mini-cards.
2. **Executive summary** — 3–4 bullets: visibility headline, PLG-funnel insight, pre-flight result as an action, sharp competitor delta.
3. **PLG Funnel by Origin** *(the business-headline section)* — rows = top origins with AI engines always listed, columns = `page_viewed` → `button_clicked` → `signup` → `activation` → `paid_conversion`. Cohort activity grouped by `gp:initial_referring_domain`. Pre-flight warning prefixed if PARTIAL/FAIL.
4. **Visibility standings** — top-6 brands and top-6 source domains from Peec.
5. **Top-cited own URLs × cite/view** — joins Peec citations with `page_viewed` on the same URLs.
6. **Recommended actions** — Action #1 is conditional on the pre-flight (tracking fix when PARTIAL/FAIL, UTM tagging when PASS). Other actions come from Peec `get_actions` drill-downs.
7. **Winners & Losers** — top prompts by SoV delta W-o-W with `get_chat` root-causes for losers.
8. **Methodology** *(collapsible)* — pre-flight rule and measured `internal_share`, cohort-vs-funnel caveat, consent caveat, per-run issues.

Real reports with production data are gitignored by default. Example reports remain in the repo (filenames matching `*-example.md`).
